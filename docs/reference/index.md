# Reference

Reference pages are the **facts**: exact shortcuts, file formats and APIs. They're
for looking things up, not for reading end-to-end.

- [Keyboard shortcuts](shortcuts.md) — every key combo, by platform.
- [Instrument telemetry API](telemetry-api.md) — the `scope()`, `meter()` and
  `plot()` functions your program calls to feed Snakie's instruments.
- [`parts.yml` reference](parts-yml.md) — every field in a Parts Library part file.
- [`robot.yml` reference](robot-yml.md) — the project file that links your board,
  parts, wiring and robot model.
- [Supported boards](supported-boards.md) — the boards Snakie can draw and flash.
- [Plugin API](plugin-api.md) — what a Snakie plugin can do.

!!! info "Reference vs. explanation"
    Reference tells you *what* something is. If you want to know *why* it works
    that way, read the matching [Explanation](../explanation/index.md) page.
